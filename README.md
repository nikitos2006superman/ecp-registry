# АИС учёта ЭЦП сотрудников Западного филиала РАНХиГС

Демонстрационная версия информационной системы учёта электронно-цифровых
подписей, разработанной в рамках дипломной работы.

## Технологический стек

- **Backend:** Python 3.10+, Flask
- **БД:** SQLite (демо) / PostgreSQL 15 (production)
- **Frontend:** HTML5, CSS3, серверный рендеринг (Jinja2)

## Установка

Требования: Python 3.10 или выше.

```bash
# 1. Распакуйте архив
unzip site_ecp_registry.zip
cd site

# 2. (опционально) Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate    # Linux/Mac
venv\Scripts\activate       # Windows

# 3. Установите зависимости
pip install flask werkzeug

# 4. Инициализируйте БД и наполните тестовыми данными
python init_db.py

# 5. Запустите сервер
python app.py
```

После запуска откройте в браузере: http://127.0.0.1:5000

## Функциональные возможности

- Реестр сотрудников филиала с привязкой к подразделениям
- Реестр сертификатов ЭП (ПЭП / НЭП / КЭП) с фильтрацией и поиском
- Реестр машиночитаемых доверенностей (МЧД)
- Реестр носителей ключевой информации (токенов)
- Журнал транзакций по токенам (выдача / возврат)
- Цветовая индикация сертификатов с истекающим сроком (≤14 дней — оранжевый, ≤7 дней — красный)
- Журнал аудита всех действий пользователей
- Ролевая модель доступа (3 роли)
- Экспорт отчётов в CSV (UTF-8 с BOM, открывается в Excel)
- Защита от CSRF, SQL-инъекций, XSS

## Различия SQLite-демо и PostgreSQL-версии

Для упрощения запуска демонстрационной версии использована SQLite (входит в
стандартную библиотеку Python). Продуктивная версия из дипломной работы
рассчитана на PostgreSQL 15, который даёт следующие преимущества:

- Поддержка одновременной многопользовательской работы без блокировок на уровне файла
- Тип JSONB для журнала аудита (в SQLite используется TEXT с JSON)
- Полнотекстовый поиск (в SQLite требует подключения расширения FTS5)
- Расширенные возможности по индексации и анализу плана запроса
- Включена в Единый реестр российского ПО

Для перехода на PostgreSQL достаточно использовать SQL-схему из Приложения 1
пояснительной записки и заменить модуль `sqlite3` на `psycopg2` либо
подключить SQLAlchemy с соответствующим драйвером.
